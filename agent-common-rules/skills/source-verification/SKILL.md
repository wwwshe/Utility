# Source Verification

Prefer verified facts over plausible memory.

## Workflow

1. Break the question into claims that need verification.
2. Prefer official documentation, release notes, standards, source repositories, or primary sources.
3. Use recent sources for facts that can change.
4. Separate facts, inferences, and recommendations.
5. Label uncertainty when sources conflict or are incomplete.
6. Provide source links for key claims.

## Avoid

- Do not invent APIs, function names, releases, prices, or policies.
- Do not present an inference as a fact.
- Do not rely on memory for current or high-stakes information.
